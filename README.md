# renderCoreHTML

Учебный проект, реализующий упрощённый HTML/CSS-рендеринг с использованием собственного layout-движка и графического backend’а на базе SFML. Проект демонстрирует послойную архитектуру, применение объектно-ориентированных принципов и паттернов проектирования, а также работу с контейнеризацией через Docker.

## Требования

Для сборки и запуска проекта требуется установленный:

- **Docker**
- (опционально) **make**

Проект не требует локальной установки компилятора или графических библиотек, так как все зависимости инкапсулированы внутри Docker-контейнера.

## Получение исходных файлов

Исходные файлы проекта можно получить одним из двух способов:

### Клонирование репозитория

```bash
git clone https://github.com/Igori452/renderCoreHTML.git
cd renderCoreHTML
```

### Загрузка архива

Также возможно скачать архив с исходными файлами и распаковать его, после чего перейти в корневую директорию проекта.

Под корневой директорией проекта понимается папка, содержащая файлы и каталоги:

src/

build/

Dockerfile

Makefile

и другие служебные файлы проекта

## Сборка проекта
С использованием Makefile (рекомендуется)

Если в системе установлена утилита make, сборку Docker-образа можно выполнить командой:
```
make build
```
Ручная сборка Docker-образа

В случае отсутствия make сборка выполняется вручную:

docker build --target full -t mini-browser:full .


⚠️ Обратите внимание: точка в конце команды является обязательной и указывает Docker на необходимость использования Dockerfile, расположенного в текущей директории.

### Запуск проекта
С использованием Makefile
```
make start
```
Ручной запуск контейнера
```
docker run -it --rm \
  -p 5900:5900 \
  -p 8080:8080 \
  -v ${PWD}:/workspace \
  mini-browser:full
```

Контейнер запускается:

в интерактивном режиме (-it);

с автоматическим удалением после завершения работы (--rm);

с пробросом портов для VNC и noVNC;

с монтированием текущей директории проекта в рабочую директорию контейнера.

Синтаксис ${PWD} корректно работает в Bash и PowerShell.
В альтернативных средах можно использовать:

$(pwd) — Bash

%cd% — Windows CMD

## Подключение к графическому интерфейсу

После запуска контейнера необходимо инициализировать графическое окружение и VNC-сервер:
```
./start-vnc.sh
```
noVNC (через браузер)

После запуска VNC-сервера графический интерфейс доступен по адресу:

http://localhost:8080/


Достаточно открыть страницу в браузере и нажать кнопку подключения.

VNC-клиент (опционально)

Также возможно прямое подключение через любой VNC-клиент по адресу:

localhost:5900


Этот способ может обеспечивать более высокое разрешение и меньшую нагрузку на систему.

## Примечания

Проект является учебным и не претендует на полную реализацию спецификаций HTML/CSS.

Архитектура проекта ориентирована на расширяемость и возможность замены графического backend’а.

Все вычисления layout выполняются внутри собственного движка, без использования сторонних браузерных библиотек.
